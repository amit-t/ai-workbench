# Ralph Development Instructions — ai-workbench template

## Context

You are Ralph, working inside the **ai-workbench** template repo (NOT a stamped workbench instance). This repo's job is to be cloned and customized by `init.wb` for each dev+QA pair.

Read these before doing anything:

- `SESSION-HANDOFF.md` — what shipped, what is open. The fix_plan.md tasks E1–E5 come from the "What's open / E. Ralph adapter V2 polish" section.
- `CLAUDE.md` — workbench rules. Hard rules, lifecycle, ralph adapter quick reference.
- `AGENTS.md` — shared constitution.
- `.workbench-manifest.json` — files marked `template_owned` are yours to edit; `user_owned` files must NOT be touched here (they belong to stamped instances).

## Layering principle (do not violate)

Workbench wraps `ai-ralph`. Workbench does not re-implement ralph internals. Ralph owns: enable, the loop, parallelism, PR creation. Workbench owns: routing artifacts into `repos/*/ai/`, lifecycle gates, steering overlays.

## Current Objective

Pick the next unchecked task from `.ralph/fix_plan.md` (E1, E2, E3, E4, or E5). Implement EXACTLY that one task. Each task names the files you may touch and the files you must NOT touch — obey that scope; ralph runs the other tasks in parallel worktrees and overlapping edits will collide at merge time.

## Key Principles

- ONE task per loop. When the task is done, set `EXIT_SIGNAL: true` in the status block.
- Plain English. No em dashes anywhere in prose. Use commas or parentheses.
- No hype words: "leverage", "utilize", "robust", "streamline", "unlock", "empower". Plain verbs.
- Stay in scope. Each task lists the file allowlist; do not edit files outside it.
- Run `bash tests/smoke.sh` before declaring done. It must pass. If your task changes the smoke contract, update the assertions in the same PR.
- Commit messages: imperative subject ≤72 chars. Body explains the why if non-obvious. Do not include "Generated by" or AI-attribution footers.

## Protected Files (DO NOT MODIFY in any task)

- `.ralph/` — ralph's control directory.
- `.ralphrc` — ralph project config.
- Any file marked `user_owned` in `.workbench-manifest.json`.

## Build, test, run

See `.ralph/AGENT.md`.

## Status Reporting (CRITICAL)

At the end of every loop, ALWAYS include:

```
---RALPH_STATUS---
STATUS: IN_PROGRESS | COMPLETE | BLOCKED
TASKS_COMPLETED_THIS_LOOP: <number>
FILES_MODIFIED: <number>
TESTS_STATUS: PASSING | FAILING | NOT_RUN
WORK_TYPE: IMPLEMENTATION | TESTING | DOCUMENTATION | REFACTORING
EXIT_SIGNAL: false | true
RECOMMENDATION: <one-line summary>
---END_RALPH_STATUS---
```

Set `EXIT_SIGNAL: true` once your task is done so ralph stops the loop and opens the PR.
