# Ralph Fix Plan — Ralph adapter V2 polish (E1–E5)

Each task below is one PR from one worktree. Tasks are scoped so file overlap is minimal; what overlap exists (CLAUDE.md, README.md) is handled by the user at merge time.

Hard rules for every task:

- Read `SESSION-HANDOFF.md`, `CLAUDE.md`, `AGENTS.md` first. Honor the layering principle (workbench wraps ralph; never re-implements ralph internals).
- Plain English. No em dashes. No hype words ("leverage", "robust", "streamline", "unlock").
- Every task starts on a worktree branch off `main`. ralph manages the worktree. Do not switch branches manually.
- Stay strictly within the file list named in your task. Other tasks own the files you are told NOT to touch; touching them will cause merge conflicts.
- Update `tests/smoke.sh` only if your change adds a new contract worth asserting (or removes one). Run `bash tests/smoke.sh` before declaring done; it must pass.
- Commit messages: imperative, ≤72 chars subject. No "Generated by" footer.
- After your single task is done, set `EXIT_SIGNAL: true` in the status block. One PR per loop.
