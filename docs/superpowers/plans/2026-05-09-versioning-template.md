# ai-workbench Versioning Template Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make every stamped workbench instance honor the versioning + upgrade-notification system. After this plan lands, every `wb.*` alias call inside a stamped wb prints an upgrade banner if the upstream `ai-workbench` template has a newer version than the version stamped into that wb.

**Architecture:**
- New `version.json` at repo root (release-please owned). This is the **template's** version.
- `aliases.sh` (template-owned, ships in every stamped wb): every `wb.*` function gets a `_wb_versioncheck wb || true;` preamble. Trivial cat-file aliases (`wb.published`, `wb.approved`, `wb.rejected`) skip the check (no actionable upgrade context during a one-shot list).
- The version-check lib itself is **not shipped from this repo**. It is dropped into `~/.local/share/wb-versioncheck/version-check.sh` by ai-devkit's `install.zsh` (and ai-ralph's `install.sh`). `aliases.sh` only sources it; if missing, the preamble is silently skipped.
- Per-stamped-wb local version is tracked in `.workbench-state/template-version.json`. That file is written by:
  - `init.wb` (in ai-devkit) at workbench creation, and
  - `wb.upgrade` (the renamed `update.wb`, in ai-devkit) after every successful template merge.
  This plan does NOT write that file; it only reads it.
- release-please workflow auto-bumps `version.json`, generates `CHANGELOG.md`, pushes git tag on every merge to `main`.
- Cache lives at `~/.cache/wb-updates/wb.json` (managed by the lib). Bootstrap-flag at `~/.cache/wb-updates/wb-bootstrapped.flag`.

**Tech Stack:** `bash` (existing aliases.sh), GitHub release-please.

## Background context (read this before starting)

### What this repo is
ai-workbench is a **GitHub template repo**. End devs never clone it globally. They run `init.wb` (from ai-devkit), which uses `gh repo create --template` to stamp a new "workbench" — a per-bundle git repo for a dev+QA pair. Files marked `template_owned` in `.workbench-manifest.json` (CLAUDE.md, scripts/, skills/, steering/, aliases.sh, …) are pulled into stamped wbs and refreshed by `wb.upgrade` (renamed from `update.wb`).

### What this plan does
Add the **template-side** of the versioning system: every stamped wb that pulls a refreshed `aliases.sh` will start firing `_wb_versioncheck wb` on every meaningful wb command, comparing the wb's `template-version.json` against upstream ai-workbench `version.json`.

### Cross-repo coordination
- The version-check **library** is NOT in this repo. It lives in `~/.local/share/wb-versioncheck/version-check.sh`, dropped there by ai-devkit's installer. If the user has not yet upgraded their ai-devkit clone, the lib is missing, and `aliases.sh` silently skips the preamble. That is the intended graceful-degradation path.
- `template-version.json` writes happen in ai-devkit's `init.wb` and `update.wb` (renamed `wb.upgrade`). This plan does NOT write it. It only consumes it via the env var `WB_TEMPLATE_VERSION_FILE`.
- This plan does not depend on ai-devkit or ai-ralph being merged first. It is dispatchable in parallel.

### Why minimal CLAUDE.md / AGENTS.md changes
Per design discussion, those files should hold only what an agent **needs to know** to behave correctly — not user docs. Long-form versioning documentation lives in ai-devkit/docs/versioning.md.

### Failure-mode summary (mirror of canonical)
- Lib missing → silently skip preamble (graceful degradation).
- All other failures handled inside the lib's fail-open path.

- [ ] **Step 3: Add the preamble helper at the top of `aliases.sh`, just after the `WB_ROOT` line**

```bash
# ── Versioning preamble helper ──────────────────────────────────────────────
# Sourced from aliases.sh by every meaningful wb.* function. Sources the
# canonical version-check lib (dropped by ai-devkit / ai-ralph installers).
# If the lib is missing, this is a silent no-op (graceful degradation for users
# who have not yet pulled the new ai-devkit installer).
_wb_check() {
  local libvc="${HOME}/.local/share/wb-versioncheck/version-check.sh"
  [[ -f "$libvc" ]] || return 0
  # shellcheck disable=SC1090
  . "$libvc"
  WB_TEMPLATE_VERSION_FILE="${WB_ROOT}/.workbench-state/template-version.json" \
    _wb_versioncheck wb 2>&1 || true
}
```

### Task 5: Add a test that verifies the preamble fires without breaking commands

**Files:**
- Create: `tests/test-aliases-preamble.sh`

- [ ] **Step 1: Write the test**

```bash
#!/usr/bin/env bash
# Verifies that:
#   1. _wb_check is defined after sourcing aliases.sh.
#   2. When the lib is missing, _wb_check is a silent no-op (graceful degradation).
#   3. When a stub lib is present, _wb_check sources it and calls _wb_versioncheck wb.
#   4. Every wrapped wb.* function still calls through to its target script (verified by
#      stubbing the target and observing it ran).
set -euo pipefail
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
REPO_ROOT="${SCRIPT_DIR}/.."

scratch="$(mktemp -d)"
trap "rm -rf '$scratch'" EXIT

# ── Case A: lib missing → preamble is silent ────────────────────────────────
HOME_OVERRIDE="$scratch/home_a"
mkdir -p "$HOME_OVERRIDE/.local/share/wb-versioncheck"
WB_FAKE_ROOT="$scratch/fake_wb_a"
mkdir -p "$WB_FAKE_ROOT/.workbench-state" "$WB_FAKE_ROOT/scripts"
# Fake target script that prints a sentinel
cat > "$WB_FAKE_ROOT/scripts/sync-context.sh" <<'SH'
#!/usr/bin/env bash
echo "SENTINEL_RAN $@"
SH
chmod +x "$WB_FAKE_ROOT/scripts/sync-context.sh"

# Source aliases.sh with WB_ROOT pointing at our fake wb
WB_ROOT_OVERRIDE="$WB_FAKE_ROOT" \
  bash -c "
    HOME='$HOME_OVERRIDE' WB_ROOT='$WB_FAKE_ROOT'
    source '$REPO_ROOT/aliases.sh'
    out=\"\$(wb.sync-context arg1 arg2 2>&1)\"
    case \"\$out\" in
      *\"SENTINEL_RAN arg1 arg2\"*) echo OK_A ;;
      *) echo \"FAIL_A: '\$out'\"; exit 1 ;;
    esac
  "

# ── Case B: lib present → _wb_versioncheck called ──────────────────────────
HOME_OVERRIDE="$scratch/home_b"
mkdir -p "$HOME_OVERRIDE/.local/share/wb-versioncheck"
cat > "$HOME_OVERRIDE/.local/share/wb-versioncheck/version-check.sh" <<'SH'
_wb_versioncheck() {
  printf "STUB_VERSIONCHECK_FIRED tool=%s template_file=%s\n" "$1" "${WB_TEMPLATE_VERSION_FILE:-}" >&2
}
SH

WB_FAKE_ROOT="$scratch/fake_wb_b"
mkdir -p "$WB_FAKE_ROOT/.workbench-state" "$WB_FAKE_ROOT/scripts"
cp "$scratch/fake_wb_a/scripts/sync-context.sh" "$WB_FAKE_ROOT/scripts/sync-context.sh"
chmod +x "$WB_FAKE_ROOT/scripts/sync-context.sh"

bash -c "
  HOME='$HOME_OVERRIDE' WB_ROOT='$WB_FAKE_ROOT'
  source '$REPO_ROOT/aliases.sh'
  out=\"\$(wb.sync-context probe 2>&1 1>/dev/null)\"
  case \"\$out\" in
    *\"STUB_VERSIONCHECK_FIRED tool=wb\"*) echo OK_B_STUB ;;
    *) echo \"FAIL_B: stub not called; out='\$out'\"; exit 1 ;;
  esac
  out2=\"\$(wb.sync-context probe 2>&1)\"
  case \"\$out2\" in
    *\"SENTINEL_RAN probe\"*) echo OK_B_PASSTHROUGH ;;
    *) echo \"FAIL_B: passthrough broke; out='\$out2'\"; exit 1 ;;
  esac
"

# ── Case C: trivial wb.published is NOT wrapped (no version-check fires) ────
WB_FAKE_ROOT="$scratch/fake_wb_c"
mkdir -p "$WB_FAKE_ROOT/.workbench-state" "$WB_FAKE_ROOT/scripts"
cat > "$WB_FAKE_ROOT/scripts/lifecycle.py" <<'PY'
#!/usr/bin/env python3
import sys
print("LIFECYCLE_RAN", *sys.argv[1:])
PY
chmod +x "$WB_FAKE_ROOT/scripts/lifecycle.py"

bash -c "
  HOME='$HOME_OVERRIDE' WB_ROOT='$WB_FAKE_ROOT'
  source '$REPO_ROOT/aliases.sh'
  out=\"\$(wb.published 2>&1)\"
  case \"\$out\" in
    *STUB_VERSIONCHECK_FIRED*) echo \"FAIL_C: trivial alias should NOT fire version-check; out='\$out'\"; exit 1 ;;
    *LIFECYCLE_RAN*list*published*) echo OK_C ;;
    *) echo \"FAIL_C: lifecycle didn't run; out='\$out'\"; exit 1 ;;
  esac
"

echo "PASS: test-aliases-preamble.sh"
```

ai-workbench (template) ships under semver. Stamped workbenches inherit the version they were created from in `.workbench-state/template-version.json`. Run `wb.upgrade` inside a stamped wb to pull template-owned files from upstream and refresh the stamp.
