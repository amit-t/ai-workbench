# Design system reference

Point to the team or shared design system used when generating screens via `/ds-screen-gen`. One block per relevant system.

## Example

- **Name:** Your team's design system (e.g. "Corp DS")
- **Repo / doc:** https://...
- **Tokens:** https://...
- **Primary components:** Button, Input, Modal, Table, Nav
- **Typical framework target:** React + TypeScript, Tailwind

## Local cache (optional)

If the design system ships NPM tokens or a storybook tarball, note the path here. `/ds-screen-gen` may pull from it.
